# Mobile-First Expense Tracking Web Application

A premium, minimalist, and lightweight expense tracking application designed for one-handed operation. It records daily expenses in 5 seconds or less and syncs them to a **Google Sheets** database via **Google Apps Script**. It also features a client-side PDF Report Generator that embeds configurable PromptPay QR payment codes.

---

## 📱 Mobile UX/UI Features

- **Thumb-Friendly Bottom Navigation**: Fixed bottom tab bar (Home, Add, Reports, Settings) designed for one-handed thumb interaction.
- **Vibrant Color Palette**: Designed with Emerald Green (`#10B981`) representing growth/wealth, and Vivid Blue (`#3B82F6`) accents for reports. Supports **Dark Mode**!
- **Fast Numeric Entry & Shortcuts**: Add expenses quickly with direct number pad inputs and shortcut buttons (`+10`, `+50`, `+100`, `+500`).
- **Interactive Reports Preview**: View summaries (Total, Average, Max, Count), interactive category breakdown progress bars, and search/filter filters.
- **Top-Layer Animating Dialogs**: Delete confirmations transition seamlessly using modern CSS `@starting-style` and native `<dialog>` element.

---

## 🛠️ Technology Stack

- **Frontend**: HTML5, Vanilla CSS3 (Custom Variables, Transitions, `@starting-style`), JavaScript ES6 modules.
- **Database**: Google Sheets.
- **Backend API**: Google Apps Script (GAS) Web App.
- **PDF Generation**: `jsPDF` library running locally in the browser (fast, offline-capable).
- **Development Tooling**: Vite for local dev server, Node.js compiler for building a single deployment-ready file.

## 📊 Database Schema (Google Sheets)

The backend script automatically initializes the following two sheets inside the spreadsheet:

### 1. `Expenses` Sheet
Stores all transactional records.
- **ID**: Unique alphanumeric identifier (`exp_timestamp_random`).
- **Timestamp**: Exact date & time of entry creation.
- **Date**: Expense transaction date (`YYYY-MM-DD`).
- **Time**: Expense transaction time (`HH:mm`).
- **Amount**: Expense numerical cost (two decimal places).
- **Category**: Associated category identifier (e.g. `food`, `travel`).
- **Description**: Optional description or note.
- **Payment Method**: Active payment choice (`QR Code`, `Cash`, or `Credit Card`).
- **Created At**: Timestamp metadata.

### 2. `Settings` Sheet
Stores configuration and user preferences.
- **Key**: Config parameter key name (e.g., `categories`, `currency`, `qr_code_mode`, `custom_qr_base64`).
- **Value**: Stored JSON string or text value.

## 🚀 Step-by-Step Deployment Guide

### 1. Setup Spreadsheet Database
1. Open the Spreadsheet you provided: [Google Sheet Database](https://docs.google.com/spreadsheets/d/14vA-VMYjO-G-ysHJ6kcUEbA3UB6dk3vm2SlC_txSzCU/edit).
2. The Apps Script backend is designed to initialize the required tables (`Expenses` and `Settings`) automatically upon your first write or sync.

### 2. Deploy Google Apps Script
1. On the spreadsheet page, click on **Extensions** -> **Apps Script**.
2. Erase any code inside `Code.gs` and paste the contents of [gas/Code.js](file:///Users/mct/My%20file%20macbook/Projects-AI/consumption-record/gas/Code.js).
3. In the Apps Script sidebar, click the **+** (Add a file) icon next to Files, select **HTML**, and name it **Index**.
4. Paste the entire contents of the compiled file [gas/Index.html](file:///Users/mct/My%20file%20macbook/Projects-AI/consumption-record/gas/Index.html) (generated by `npm run build`) into this file, replacing the default boilerplate.
5. In the Apps Script project editor, click **Save** (disk icon).

### 3. Publish Web Application
1. Click **Deploy** in the top right corner and select **New deployment**.
2. Click the gear icon next to "Select type" and choose **Web app**.
3. Configure the settings:
   - **Description**: `Expense Tracking App v1.0`
   - **Execute as**: `Me (your email)`
   - **Who has access**: `Anyone`
4. Click **Deploy**.
5. Copy the **Web App URL** provided. This is the link you will bookmark on your mobile home screen to run the app.

## 💻 Local Development & Prototyping

To run the application locally for testing and debugging, run:

```bash
# 1. Install Vite development tool
npm install

# 2. Start the local server
npm run dev
```

This starts a server at `http://localhost:3000` and automatically opens it in your browser. 
- When opened locally, the application recognizes it is not in the Google Apps Script environment and runs in **Mock Mode** using browser `localStorage` as a mock database.
- You can fully add, edit, delete expenses, search, filter, preview reports, configure categories, upload custom QR codes, and generate/download PDFs locally!
- To build the compiled single-file, run:
  ```bash
  npm run build
  ```
